## 0. Changelog

### v3.1 Changes from v3.0

- **[CRITICAL] Universal graceful truncation added to all LLM output models.** Introduced `TruncatingModel`, a shared Pydantic `BaseModel` subclass in `models.py`. It carries a `@model_validator(mode='before')` that inspects every string field for a declared `max_length` constraint and silently truncates the value to that limit before Pydantic runs field-level validation. `CandidateVerdict`, `ImpactedItem`, and `ImpactReport` now inherit from `TruncatingModel` instead of `BaseModel`. **Motivation:** LLMs cannot reliably count characters; strict `max_length` constraints were causing `ValidationError` pipeline crashes (observed in Step 4, LLM Call #2 `CandidateVerdict.justification`). The architectural rule is stated below in §2 and applies to all future LLM output schemas in this project.

### v3.0 Changes from v2.0 (Chapter III Alignment Audit)

- **[CRITICAL] CRInterpretation schema updated.** Added `is_actionable` (bool) and `rejection_reason` (str, optional) fields per Subbab III.2.2.1 GIGO validation design. LLM Call #1 now performs feasibility check before semantic extraction. If is_actionable is false, pipeline halts early and returns rejection report without executing Calls #2 and #3.
- **[CRITICAL] interpreter.py updated.** System prompt now instructs LLM to assess CR feasibility first. Returns CRInterpretation with is_actionable field.
- **[CRITICAL] runner.py updated.** Added GIGO early termination checkpoint between Step 1 and Step 2. If cr_interp.is_actionable is False, pipeline returns a minimal rejection JSON and exits.
- **[MINOR] Title updated.** Document renamed from "Revised Developer & AI Implementation Blueprint" to "Master Implementation Blueprint."

### v2.0 Changes from v1.0

- **[D1] Single-collection retriever replaced with dual-path orchestrator.** `hybrid_search()` now accepts two ChromaDB collection handles and two BM25 indices. Each query produces four ranked lists (dense-doc, BM25-doc, dense-code, BM25-code). All four fuse via a single RRF pass. Previous design silently halved the retrieval surface.
- **[D2] `affected_layers` → `chunk_type` mapping layer added.** `CRInterpretation.affected_layers` values (`requirement`, `design`, `code`) are now mapped to ChromaDB filter values (`FR`, `NFR`, `Design`) through an explicit translation dict. `code` activates the code collection path, not a chunk_type filter. Previous design filtered against non-existent chunk_type values and returned zero results.
- **[D3] CIS evaluation sort key changed from severity to structural edge weight.** Severity (`Tinggi`/`Menengah`/`Rendah`) is still generated by LLM Call #3 for human consumption in the report, but evaluation ranking uses `min(EDGE_WEIGHT[e] for e in causal_chain)` as the second sort key. This aligns with the locked III.4.3.4 revision.
- **[D4] BFS return value now includes seed nodes.** `bfs_propagate()` returns a `CISResult` dataclass with explicit `sis_nodes` and `propagated_nodes` dicts plus a `combined()` method. Prevents silent exclusion of SIS nodes from Precision@K computation.
- **[D5] Lost-in-the-middle ordering rewritten.** Previous `insert(-1, c)` loop produced descending middle order (opposite of specification). Replaced with explicit `sorted(middle, key=rrf_score, ascending)` then concatenation.
- **[D6] Doc-to-code seed resolution added.** New `resolve_sis_to_code_seeds()` function in `runner.py` resolves doc chunk SIS nodes to their top-K associated code nodes via `doc_code_candidates` table before BFS invocation. Previous design silently dropped doc-chunk seeds (zero edges in code graph).
- **[NEW] Cross-encoder reranker integrated.** `BAAI/bge-reranker-v2-m3` via `FlagEmbedding.FlagReranker` inserted between RRF fusion and LLM Call #2. Reranker scores each `(CR_intent, candidate_text)` pair and re-sorts candidates by cross-encoder relevance before LLM validation. This adds a precision layer that reduces false positives entering the expensive LLM call.
- **[NEW] Deterministic LLM constraints hardcoded.** All three LLM calls set `temperature=0` and `seed=42` (configurable via env). Caveat documented: OpenAI's `seed` parameter is best-effort, not bitwise-deterministic across model versions.
- **[NEW] Context window overflow guard for LLM Call #3.** Token budget computed via `tiktoken` before synthesis call. If CIS exceeds budget, nodes are truncated by reverse structural weight (lowest-significance nodes dropped first) until payload fits. Truncation is logged.
- **[NEW] Arrow function extraction added to AST parser.** `lexical_declaration` → `variable_declarator` → `arrow_function` pattern handled. React component classification applies the same uppercase-name + JSX-body heuristic. This captures the dominant function declaration pattern in Next.js codebases.
- **[NEW] Re-export and namespace import handling.** `export { X } from './module'` and `import * as X from './module'` patterns added to IMPORTS edge extraction.

## 2. Data Models (Pydantic v2 Schemas)

> **Architectural Rule — LLM String Truncation (v3.1):**
> All LLM-generated string fields **must** utilise graceful truncation fallback
> mechanisms to prevent pipeline crashes from Pydantic strict length limits.
> LLMs cannot reliably count characters; rather than raising `ValidationError`
> when `max_length` is exceeded, every LLM output schema MUST inherit from
> `TruncatingModel` (see below). The `@model_validator(mode='before')` in that
> base class silently truncates any overlong string to its declared limit before
> field-level validation runs. Future LLM output schemas added to this project
> MUST also inherit from `TruncatingModel`.

```python
# models.py
from __future__ import annotations
from pydantic import BaseModel, Field, model_validator
from typing import Any, Literal
from dataclasses import dataclass, field

# ── Shared Base Model (v3.1) ─────────────────────────────────────────

class TruncatingModel(BaseModel):
    """Base model with graceful LLM output string truncation.

    Silently truncates any string field that exceeds its declared max_length
    before Pydantic runs field-level validation, preventing ValidationError
    crashes caused by LLMs that cannot reliably count characters.
    """
    @model_validator(mode='before')
    @classmethod
    def _truncate_overlong_strings(cls, data: Any) -> Any:
        if not isinstance(data, dict):
            return data
        for field_name, field_info in cls.model_fields.items():
            value = data.get(field_name)
            if not isinstance(value, str):
                continue
            for meta in field_info.metadata:
                max_len = getattr(meta, 'max_length', None)
                if max_len is not None and len(value) > max_len:
                    data[field_name] = value[:max_len]
                    break
        return data


# ── LLM Call #1: CR Interpretation ──────────────────────────────────

class CRInterpretation(BaseModel):
    is_actionable: bool = Field(
        description="False if CR is too ambiguous or contains no identifiable change intent."
    )
    rejection_reason: str | None = Field(
        default=None,
        description="One sentence explaining why CR was rejected. Null if is_actionable is True."
    )
    primary_intent: str = Field(
        description="Single sentence: what is being changed and why."
    )
    change_type: Literal["add", "modify", "remove"]
    affected_layers: list[Literal["requirement", "design", "code"]]
    affected_domain_concepts: list[str] = Field(
        description="Business domain concepts, explicit and implied.",
        min_length=1, max_length=10
    )
    search_queries: list[str] = Field(
        description=(
            "2-3 English technical phrases optimized for vector search "
            "against code function signatures and doc section titles."
        ),
        min_length=2, max_length=3
    )


# ── LLM Call #2: SIS Validation ────────────────────────────────────

class CandidateVerdict(TruncatingModel):  # truncates function_purpose/mechanism_of_impact/justification
    node_id: str = Field(description="The candidate's node_id, copied verbatim.")
    function_purpose: str = Field(max_length=150, description="One sentence: what this node does.")
    mechanism_of_impact: str = Field(max_length=200, description="Concrete modification mechanism, or EMPTY to reject.")
    justification: str = Field(max_length=200, description="One-sentence confirmation/rejection summary.")
    confirmed: bool = Field(description="True only if mechanism_of_impact is non-empty and structural.")

class SISValidationResult(BaseModel):
    verdicts: list[CandidateVerdict]


# ── LLM Call #3: Impact Report ─────────────────────────────────────

class ImpactedItem(TruncatingModel):  # TruncatingModel: truncates structural_justification if > 200 chars
    node_id: str
    node_type: str
    file_path: str
    severity: Literal["Tinggi", "Menengah", "Rendah"]
    causal_chain: list[str] = Field(
        description="Ordered edge types from SIS root to this node."
    )
    structural_justification: str = Field(max_length=200)
    traceability_backlinks: list[str] = Field(default_factory=list)

class ImpactReport(TruncatingModel):  # TruncatingModel: truncates executive_summary if > 800 chars
    executive_summary: str = Field(max_length=800)
    impacted_items: list[ImpactedItem]
    requirement_conflicts: list[str] = Field(default_factory=list)
    estimated_change_scope: Literal["terlokalisasi", "menengah", "ekstensif"]


# ── BFS Output ─────────────────────────────────────────────────────

@dataclass
class NodeTrace:
    depth: int
    causal_chain: list[str]
    path: list[str]
    source_seed: str

@dataclass
class CISResult:
    sis_nodes: dict[str, NodeTrace] = field(default_factory=dict)
    propagated_nodes: dict[str, NodeTrace] = field(default_factory=dict)

    def combined(self) -> dict[str, NodeTrace]:
        return {**self.sis_nodes, **self.propagated_nodes}

    def all_node_ids(self) -> list[str]:
        return list(self.sis_nodes.keys()) + list(self.propagated_nodes.keys())


# ── Evaluation Sort ────────────────────────────────────────────────

EDGE_WEIGHT: dict[str, int] = {
    "IMPLEMENTS": 0, "TYPED_BY": 0,
    "CALLS": 1, "INHERITS": 1, "DEFINES_METHOD": 1,
    "IMPORTS": 2, "RENDERS": 2, "DEPENDS_ON_EXTERNAL": 2,
}

def structural_weight(causal_chain: list[str]) -> int:
    """Lower value = higher significance. SIS seeds return -1 (sort first)."""
    if not causal_chain:
        return -1
    return min(EDGE_WEIGHT.get(e, 2) for e in causal_chain)
```

**Schema design notes for BFS traversal:**

The `idx_edges_target` index is critical. BFS reverse traversal queries `WHERE target_id = ? AND edge_type = ?` to find all predecessors. Without this index, every BFS hop scans the full edge table. With it: O(log E) per lookup. For Haidar (2025) scale (~2000 edges), this is negligible, but the index prevents degradation on larger repos.

`source_code` column stores full function/method body text. This is NOT embedded — it is injected into LLM Call #3 context for impacted nodes where the synthesizer needs to explain structural justification. Stored separately from `embed_text` to enforce the thesis design: only signatures + docstrings are embedded.

### 6.2 `indexer/code_indexer.py`

**Input:** Repository root path.  
**Output:** `code_nodes` and `structural_edges` tables populated in SQLite.

```python
#                                                   contains jsx_element → classified
#                                                   as React component (still Function
#                                                   node_type, flagged in metadata)

# class_declaration              → Class            extends = INHERITS, implements = IMPLEMENTS
# interface_declaration          → Interface
# type_alias_declaration         → TypeAlias
# enum_declaration               → Enum
# ─────────────────────────────────────────────────────────────────

# ── Edge extraction ────────────────────────────────────────────────
#
# Edge type           Source of truth in AST
# ─────────────────────────────────────────────────────────────────
# CALLS               call_expression inside function/method body.
#                     Target = resolved function identifier.
#                     FILTER: skip builtins (console.*, Object.*, Array.*,
#                     Math.*, JSON.*, Promise.*, setTimeout, setInterval,
#                     parseInt, parseFloat, String, Number, Boolean, Error).

#
```

### 6.4 `indexer/reranker.py`

```python
from FlagEmbedding import FlagReranker

class Reranker:
    def __init__(self, model_name: str = "BAAI/bge-reranker-v2-m3"):
        self.model = FlagReranker(model_name, use_fp16=True)

    def rerank(
        self,
        query: str,
        candidates: list[dict],     # [{node_id, text_snippet, rrf_score, ...}]
        top_k: int = 15
    ) -> list[dict]:
        """
        Score each (query, candidate.text_snippet) pair with cross-encoder.
        Return candidates re-sorted by cross-encoder score, truncated to top_k.
        """
        if not candidates:
            return []

        pairs = [(query, c["text_snippet"]) for c in candidates]
        scores = self.model.compute_score(pairs, normalize=True)

        # Attach score, sort descending
        for c, s in zip(candidates, scores):
            c["reranker_score"] = float(s)

        ranked = sorted(candidates, key=lambda c: c["reranker_score"], reverse=True)
        return ranked[:top_k]
```

**Position in pipeline:** After RRF fusion (which outputs ≤15 candidates), before LLM Call #2 validation. The reranker provides a more precise relevance signal than cosine similarity alone, especially for cross-lingual query-document pairs where the embedding model's dot-product approximation may be noisy.

**Complexity:** O(N) forward passes through a ~560M parameter cross-encoder. For 15 candidates: ~2-3 seconds on CPU, <0.5s on GPU.

### 6.5 `indexer/traceability.py`

```python
import numpy as np

def compute_doc_code_candidates(
    code_vecs: dict[str, np.ndarray],   # node_id → (D,) vector
    doc_vecs: dict[str, np.ndarray],    # chunk_id → (D,) vector
    top_k: int = 5
) -> list[tuple[str, str, float]]:
    """
    Brute-force cosine similarity. Returns (code_id, doc_id, similarity).
    Complexity: O(N_code × N_doc × D) for matrix multiply.
    At Haidar scale (500 × 100 × 1024): ~50M FLOPs → sub-second.
    """
    code_ids = list(code_vecs.keys())
    doc_ids = list(doc_vecs.keys())

    C = np.stack([code_vecs[cid] for cid in code_ids])    # (N_code, D)
    D = np.stack([doc_vecs[did] for did in doc_ids])       # (N_doc, D)

    # L2-normalize rows for cosine similarity via dot product
    C = C / (np.linalg.norm(C, axis=1, keepdims=True) + 1e-10)
    D = D / (np.linalg.norm(D, axis=1, keepdims=True) + 1e-10)

    sim = C @ D.T   # (N_code, N_doc)

    results = []
    for i, cid in enumerate(code_ids):
        top_j = np.argsort(sim[i])[::-1][:top_k]
        for j in top_j:
            results.append((cid, doc_ids[j], float(sim[i, j])))

    return results
```

### 6.6 `pipeline/retriever.py`

```python
import numpy as np
from rank_bm25 import BM25Okapi

# ── affected_layers → chunk_type mapping ────────────────────────────
LAYER_TO_CHUNK_TYPES: dict[str, list[str]] = {
    "requirement": ["FR", "NFR"],
    "design":      ["Design"],
    "code":        [],  # code search goes to code_collection, not filtered by chunk_type
}

def resolve_doc_filter(affected_layers: list[str]) -> list[str] | None:
    """Convert affected_layers to chunk_type values for doc collection filter."""
    types = []
    for layer in affected_layers:
        types.extend(LAYER_TO_CHUNK_TYPES.get(layer, []))
    return types if types else None

def should_search_code(affected_layers: list[str]) -> bool:
    return "code" in affected_layers


def reciprocal_rank_fusion(
    ranked_lists: list[list[str]],
    k: int = 60
) -> dict[str, float]:
    """RRF over N ranked lists. Returns {node_id: rrf_score}."""
    scores: dict[str, float] = {}
    for rlist in ranked_lists:
        for rank, nid in enumerate(rlist):
            scores[nid] = scores.get(nid, 0.0) + 1.0 / (k + rank + 1)
    return scores


def hybrid_search(
    queries: list[str],
    doc_collection,                     # ChromaDB collection: doc_chunks
    code_collection,                    # ChromaDB collection: code_units
    doc_bm25: BM25Okapi,
    doc_bm25_ids: list[str],
    code_bm25: BM25Okapi,
    code_bm25_ids: list[str],
    embedder,
    affected_layers: list[str],
    top_k_per_query: int = 15,
    rrf_k: int = 60,
    max_candidates: int = 15,
) -> list[dict]:
    """
    Dual-path hybrid search. Returns list of candidate dicts
    sorted by RRF score descending, truncated to max_candidates.
    
    Four ranked lists per query:
      1. Dense-doc:  cosine search on doc_chunks
      2. BM25-doc:   lexical search on doc chunk texts
      3. Dense-code: cosine search on code_units
      4. BM25-code:  lexical search on code unit texts
    
    All lists across all queries fused via single RRF pass.
    """
    all_ranked_lists: list[list[str]] = []
    doc_filter = resolve_doc_filter(affected_layers)
    search_code = should_search_code(affected_layers)

    for query in queries:
        query_vec = embedder.embed_single(query)

        # ── Doc path ──
        where_clause = {"chunk_type": {"$in": doc_filter}} if doc_filter else None
        doc_dense = doc_collection.query(
            query_embeddings=[query_vec],
            n_results=top_k_per_query,
            where=where_clause,
        )
        all_ranked_lists.append(doc_dense["ids"][0])

        tokenized_q = query.lower().split()
        doc_bm25_scores = doc_bm25.get_scores(tokenized_q)
        doc_bm25_ranked = [
            doc_bm25_ids[i]
            for i in np.argsort(doc_bm25_scores)[::-1][:top_k_per_query]
        ]
        all_ranked_lists.append(doc_bm25_ranked)

        # ── Code path (only if affected_layers includes code) ──
        if search_code:
            code_dense = code_collection.query(
                query_embeddings=[query_vec],
                n_results=top_k_per_query,
            )
            all_ranked_lists.append(code_dense["ids"][0])

            code_bm25_scores = code_bm25.get_scores(tokenized_q)
            code_bm25_ranked = [
                code_bm25_ids[i]
                for i in np.argsort(code_bm25_scores)[::-1][:top_k_per_query]
            ]
            all_ranked_lists.append(code_bm25_ranked)

    rrf_scores = reciprocal_rank_fusion(all_ranked_lists, k=rrf_k)
    sorted_candidates = sorted(rrf_scores.items(), key=lambda x: x[1], reverse=True)

    # Build candidate dicts with text snippets for reranker
    # Fetch text from ChromaDB or SQLite as needed
    results = []
    for nid, score in sorted_candidates[:max_candidates]:
        results.append({
            "node_id": nid,
            "rrf_score": score,
            "text_snippet": "",   # populated by caller from ChromaDB/SQLite
            "node_type": "",      # populated by caller
        })
    return results


def build_bm25_index(
    texts: list[str], ids: list[str]
) -> tuple[BM25Okapi, list[str]]:
    """
    Rebuild per session. ~0.5s for 300 documents. No persistence.
    """
    tokenized = [text.lower().split() for text in texts]
    return BM25Okapi(tokenized), ids
```

### 6.7 `pipeline/graph_bfs.py`

```python
import networkx as nx
from collections import deque
from models import NodeTrace, CISResult

EDGE_CONFIG: dict[str, dict] = {
    "CALLS":               {"direction": "reverse", "max_depth": 3},
    "INHERITS":            {"direction": "reverse", "max_depth": 3},
    "IMPLEMENTS":          {"direction": "reverse", "max_depth": 3},
    "TYPED_BY":            {"direction": "reverse", "max_depth": 3},
    "DEFINES_METHOD":      {"direction": "forward", "max_depth": 3},
    "IMPORTS":             {"direction": "reverse", "max_depth": 1},
    "DEPENDS_ON_EXTERNAL": {"direction": "reverse", "max_depth": 1},
    "RENDERS":             {"direction": "reverse", "max_depth": 1},
}

def build_graph_from_sqlite(conn) -> nx.MultiDiGraph:
    """Load full edge set into NetworkX. Called once at pipeline startup."""
    G = nx.MultiDiGraph()
    rows = conn.execute(
        "SELECT source_id, target_id, edge_type FROM structural_edges"
    ).fetchall()
    for src, tgt, etype in rows:
        G.add_edge(src, tgt, edge_type=etype)
    return G


def bfs_propagate(
    graph: nx.MultiDiGraph,
    seed_node_ids: list[str],
) -> CISResult:
    """
    Multi-seed BFS with per-edge-type direction and depth limits.
    
    Returns CISResult with:
      - sis_nodes: seed nodes at depth 0
      - propagated_nodes: discovered nodes at depth 1..max_depth
    
    Edge direction semantics:
      "reverse" = impact flows from target to sources (predecessors).
        Edge A→B means A depends on B. If B changes, A is impacted.
        Query: graph.in_edges(B) to find all A.
      "forward" = impact flows from source to targets (successors).
        Edge A→B means A owns B (DEFINES_METHOD only).
        If A changes, B is impacted. Query: graph.out_edges(A).
    """
    result = CISResult()
    visited: set[str] = set()

    # Initialize seeds
    for seed in seed_node_ids:
        visited.add(seed)
        result.sis_nodes[seed] = NodeTrace(
            depth=0, causal_chain=[], path=[seed], source_seed=seed
        )

    # BFS queue: (current_node, current_depth, chain_so_far, path_so_far, origin_seed)
    queue: deque[tuple[str, int, list[str], list[str], str]] = deque()
    for seed in seed_node_ids:
        queue.append((seed, 0, [], [seed], seed))

    while queue:
        node, depth, chain, path, origin = queue.popleft()

        for edge_type, cfg in EDGE_CONFIG.items():
            if depth >= cfg["max_depth"]:
                continue

            # Get neighbors based on direction
            if cfg["direction"] == "reverse":
                # A→B in graph, B is current node → find all A
                neighbors = [
                    u for u, _, d in graph.in_edges(node, data=True)
                    if d.get("edge_type") == edge_type
                ]
            else:
                # A→B in graph, A is current node → find all B
                neighbors = [
                    v for _, v, d in graph.out_edges(node, data=True)
                    if d.get("edge_type") == edge_type
                ]

            for nbr in neighbors:
                if nbr in visited:
                    continue
                visited.add(nbr)

                new_depth = depth + 1
                new_chain = chain + [edge_type]
                new_path = path + [nbr]

                result.propagated_nodes[nbr] = NodeTrace(
                    depth=new_depth,
                    causal_chain=new_chain,
                    path=new_path,
                    source_seed=origin,
                )
                queue.append((nbr, new_depth, new_chain, new_path, origin))

    return result
```

**Correctness invariant:** `len(result.sis_nodes) + len(result.propagated_nodes) == len(visited)`. Assert this in tests.

**Complexity:** O(V + E) where V = reachable nodes, E = edges traversed. Bounded by depth limits: worst case with global depth 3 on a fully connected graph is O(d^3) where d = average out-degree. For Haidar scale: <1ms.

```python
import sqlite3

def resolve_sis_to_code_seeds(
    sis_node_ids: list[str],
    conn: sqlite3.Connection,
    top_k: int = 3,
) -> tuple[list[str], dict[str, list[str]]]:
    """
    Resolve SIS containing mixed doc/code node IDs to pure code node IDs.
    
    Returns:
      - code_seeds: deduplicated list of code node IDs for BFS
      - doc_to_code_map: {doc_chunk_id: [resolved_code_ids]} for traceability
    """
    code_seeds: list[str] = []
    doc_to_code_map: dict[str, list[str]] = {}

    code_node_ids = set(
        r[0] for r in conn.execute("SELECT node_id FROM code_nodes").fetchall()
    )

    for nid in sis_node_ids:
        if nid in code_node_ids:
            code_seeds.append(nid)
        else:
            # Doc chunk → resolve via traceability table
            rows = conn.execute(
                "SELECT code_id FROM doc_code_candidates "
                "WHERE doc_id = ? ORDER BY similarity DESC LIMIT ?",
                (nid, top_k),
            ).fetchall()
            resolved = [r[0] for r in rows]
            doc_to_code_map[nid] = resolved
            code_seeds.extend(resolved)

    # Deduplicate preserving order
    seen = set()
    deduped = []
    for cid in code_seeds:
        if cid not in seen:
            seen.add(cid)
            deduped.append(cid)

    return deduped, doc_to_code_map
```

```python
def mitigate_lost_in_middle(candidates: list[dict]) -> list[dict]:
    """
    Reorder for lost-in-the-middle mitigation.
```

### 6.11 `pipeline/context_builder.py` — Token-Budgeted Context for Call #3

```python
import tiktoken
from models import CISResult, CRInterpretation, structural_weight

def estimate_tokens(text: str, model: str = "gpt-4o-mini") -> int:
    enc = tiktoken.encoding_for_model(model)
    return len(enc.encode(text))

def build_synthesis_context(
    cr_text: str,
    cr_interp: CRInterpretation,
    cis: CISResult,
    sis_rrf_scores: dict[str, float],
    backlinks: dict[str, list[tuple[str, float]]],
    code_snippets: dict[str, str],
    settings,
) -> str:
    """
    Assemble context payload for LLM Call #3.
    Truncates CIS nodes if total tokens exceed budget.
    Truncation order: lowest structural weight (least significant) dropped first.
    """
    budget = (
        settings.llm_max_context_tokens
        - settings.synthesis_system_prompt_tokens
        - 2000  # reserve for output tokens
    )

    # Fixed context: CR + interpretation (always included)
    header = (
        f"## Change Request\n{cr_text}\n\n"
        f"## Interpreted Intent\n{cr_interp.primary_intent}\n"
        f"Change type: {cr_interp.change_type}\n"
        f"Affected layers: {', '.join(cr_interp.affected_layers)}\n"
        f"Domain concepts: {', '.join(cr_interp.affected_domain_concepts)}\n\n"
    )
    header_tokens = estimate_tokens(header)
    remaining = budget - header_tokens

    # Build per-node blocks, sorted by priority (most significant first)
    all_nodes = cis.combined()
    node_blocks: list[tuple[str, str, int]] = []  # (node_id, block_text, token_count)

    for nid, trace in all_nodes.items():
        block = f"### {nid}\n"
        block += f"Depth: {trace.depth} | Chain: {' → '.join(trace.causal_chain) or 'SIS seed'}\n"
        block += f"Path: {' → '.join(trace.path)}\n"
        if nid in code_snippets:
            sig = code_snippets[nid][:500]  # hard cap per-node source
            block += f"Signature:\n```\n{sig}\n```\n"
        if nid in backlinks:
            links = backlinks[nid][:3]  # top-3 backlinks per node
            link_str = "; ".join(f"{did} (sim={sim:.3f})" for did, sim in links)
            block += f"Backlinks: {link_str}\n"
        block += "\n"

        tokens = estimate_tokens(block)
        node_blocks.append((nid, block, tokens))

    # Sort by priority: SIS seeds first (depth 0), then by structural weight ascending
    node_blocks.sort(key=lambda x: (
        all_nodes[x[0]].depth,
        structural_weight(all_nodes[x[0]].causal_chain),
    ))

    # Greedily include nodes until budget exhausted
    included_blocks: list[str] = []
    truncated_count = 0
    for nid, block, tok in node_blocks:
        if remaining - tok < 0:
            truncated_count += 1
            continue
        included_blocks.append(block)
        remaining -= tok

    context = header + "## Impacted Elements\n\n" + "".join(included_blocks)

    if truncated_count > 0:
        context += (
            f"\n[NOTE: {truncated_count} lower-priority nodes truncated "
            f"due to context window limit.]\n"
        )

    return context
```

**Overflow strategy:** Nodes are sorted by `(depth, structural_weight)`. SIS seeds and high-significance relationships are always retained. Low-significance nodes at depth 3 via IMPORTS/RENDERS are dropped first. This preserves the most causally critical information. The truncation count is injected into context so the LLM can note it in the report's `executive_summary`.

```python
            executive_summary=f"CR rejected: {cr_interp.rejection_reason}",
            impacted_items=[],
            requirement_conflicts=[],
            estimated_change_scope="terlokalisasi",
        )

    # ── Step 2: Dual-path hybrid search + RRF ──
    candidates = hybrid_search(
        queries=cr_interp.search_queries,
        doc_collection=doc_col,
        code_collection=code_col,
        doc_bm25=doc_bm25, doc_bm25_ids=doc_bm25_ids,
        code_bm25=code_bm25, code_bm25_ids=code_bm25_ids,
        embedder=embedder,
        affected_layers=cr_interp.affected_layers,
        rrf_k=settings.rrf_k,
        max_candidates=settings.max_candidates_post_rrf,
    )
    # Populate text_snippet and node_type from stores
    candidates = enrich_candidates(candidates, doc_col, code_col, conn)
    logger.info("Hybrid search: {} candidates post-RRF", len(candidates))

    # ── Step 3: Cross-encoder reranking ──
    rerank_query = cr_interp.primary_intent
    candidates = reranker.rerank(rerank_query, candidates,
                                  top_k=settings.max_candidates_post_rerank)
    logger.info("Reranked: {} candidates", len(candidates))

    # ── Step 4: LLM Call #2 — Validate SIS ──
    validation = validate_sis_candidates(cr_interp, candidates, settings)
    sis_node_ids = [
        v.node_id for v in validation.verdicts if v.confirmed
    ]
    sis_rrf_scores = {
        c["node_id"]: c["rrf_score"] for c in candidates
        if c["node_id"] in set(sis_node_ids)
    }
    logger.info("SIS validated: {}/{} confirmed", len(sis_node_ids), len(candidates))

    # ── Step 5: Resolve doc SIS → code seeds ──
    code_seeds, doc_code_map = resolve_sis_to_code_seeds(
        sis_node_ids, conn, top_k=settings.top_k_traceability
    )
    logger.info("Code seeds for BFS: {} (from {} SIS nodes)", len(code_seeds), len(sis_node_ids))

    # ── Step 6: BFS propagation (deterministic, no LLM) ──
    cis = bfs_propagate(graph, code_seeds)
    logger.info("CIS: {} SIS + {} propagated = {} total",
                len(cis.sis_nodes), len(cis.propagated_nodes),
                len(cis.all_node_ids()))

    # ── Step 7: Fetch backlinks + code snippets ──
    backlinks = fetch_backlinks(cis.all_node_ids(), conn)
    code_snippets = fetch_code_snippets(cis.all_node_ids(), conn)

    # ── Step 8: Build token-budgeted context ──
    context = build_synthesis_context(
        cr_text, cr_interp, cis, sis_rrf_scores,
        backlinks, code_snippets, settings
    )
    logger.info("Synthesis context: ~{} tokens", estimate_tokens(context))

    # ── Step 9: LLM Call #3 — Synthesize report ──
    report = synthesize_report(context, settings)
    logger.info("Report generated. {} impacted items, scope: {}",
                len(report.impacted_items), report.estimated_change_scope)

    return report
```

## 7. Evaluation Pipeline

### 7.1 CIS Sorting for Metric Computation

```python
from models import structural_weight, CISResult

def sort_cis_for_evaluation(
    cis: CISResult,
    sis_rrf_scores: dict[str, float],
) -> list[str]:
    """
    Returns ordered list of node_ids for Precision@K / Recall@K computation.
    
    Sort key hierarchy (all ascending):
      1. BFS depth (0 = SIS seed, first)
      2. structural_weight (0 = IMPLEMENTS/TYPED_BY, most significant)
      3. -rrf_score of source seed (higher RRF = earlier)
    """
    all_nodes = cis.combined()
    
    def sort_key(nid: str):
        trace = all_nodes[nid]
        return (
            trace.depth,
            structural_weight(trace.causal_chain),
            -sis_rrf_scores.get(trace.source_seed, 0.0),
        )
    
    return sorted(all_nodes.keys(), key=sort_key)
```
